What Is QR Pro Scanner?

QR Pro Multi-Instrument Scanner is the companion tool to Quantum Ribbon Pro. It runs the full QR Pro signal engine across up to 20 symbols and timeframes simultaneously, displaying live trade status in a single dashboard table — without switching charts.

If you're already using Quantum Ribbon Pro on individual charts, this gives you market-wide visibility using the exact same logic, risk management, and alert system.

Companion Tool

The Scanner is designed to complement Quantum Ribbon Pro, not replace it. For the best workflow, use the Scanner to monitor your watchlist, then switch to a chart with the main indicator to review signals in detail before executing.

You can also use it as a standalone scanner if you want multi-instrument monitoring without opening individual charts at all — all the signal logic runs internally.

Setup & Getting Started

After purchase, you'll receive access to the Scanner as an invite-only script on TradingView, just like the main indicator. It will appear in your Invite-Only Scripts folder.

To add it to your chart:

  1. Open any chart on TradingView — the underlying chart symbol doesn't matter for the scanner
  2. Open the Indicators panel and go to Invite-Only Scripts
  3. Select QR Pro Multi-Instrument Scanner
  4. The dashboard table will appear on your chart immediately

Which chart should I put it on?

Since the scanner pulls data from its own symbol inputs, the chart you place it on doesn't affect results. Many traders use a low-timeframe chart like a 1-minute or 5-minute chart as their "scanner chart" to ensure the scanner updates frequently.

Once added, open the indicator settings to configure your symbols, timeframes, and signal preferences.

Adding Symbols & Timeframes

The scanner supports up to 20 instruments. Each row has three inputs: an enable checkbox, a symbol field, and a timeframe selector.

The first 5 symbols are enabled by default with example instruments (EURUSD, BTCUSDT, SPX, Gold, RHM). Replace these with your own watchlist. Symbols 6–20 are disabled by default — check the enable box to activate each one.

Entering symbols: Use TradingView's full symbol format including the exchange prefix. For example:

  • FX:EURUSD — Forex via FX data feed
  • BYBIT:BTCUSDT.P — Bybit perpetual futures
  • NASDAQ:AAPL — US stocks
  • CAPITALCOM:GOLD — Gold CFD
  • SP:SPX — S&P 500 index

Timeframe field: Leave blank to use the chart's current timeframe. Enter a value like 15, 60, 240, or D to lock that row to a specific timeframe regardless of what chart you're on.

⚠ Timeframe Limitation

The scanner timeframe for each symbol cannot be lower than the chart's current timeframe. If your chart is on 1H, you cannot scan 15M timeframes for any symbol. Set the chart to a lower timeframe if you need to monitor shorter intervals.

Position Column

Shows the current trade state for that instrument. LONG or SHORT means an active trade is open. WAIT means no position is active — the scanner is watching for a signal.

When a trade is active, additional indicators appear in brackets: (BE) means break-even stop has been moved to entry. (TP1✓) or (TP2✓) shows which partial take-profit levels have been hit.

Master Trend Column

Shows overall ribbon alignment for that symbol. The five possible states are Strong Bull, Bullish, Neutral, Bearish, and Strong Bear. When the range filter is active and blocking signals, it shows Filtered / Range instead.

Confidence Column

A market health score from 0–100% based on ribbon alignment strength, trend consistency, and volume participation. Higher percentages indicate stronger trending conditions. This is a context indicator — not a signal trigger by itself.

Entry / SL / TP Columns

Live price levels for the active trade. Entry shows the exact price where the position was opened. SL shows the current stop level (this updates if break-even automation moves it). TP shows the final take-profit target. These are blank when no position is open.

Table Position & Size

You can reposition the table to any corner of the chart (top right, bottom left, etc.) and adjust text size from tiny to large via the Scanner Layout settings. Customize it to fit your screen setup.

Signal Engine Settings

The scanner uses the same signal logic as the main Quantum Ribbon Pro indicator. All signal settings apply globally — the same configuration runs across every symbol in your watchlist.

Signal Sensitivity

Controls how selective the signal engine is. Scale from 1 (most signals) to 10 (fewest signals). Higher sensitivity adds a longer cooldown between signals and requires stronger confirmation. Default is 5 — a balanced starting point for most markets.

Ribbon Bounce Signals

When disabled (default), signals fire on trend confirmation breakouts. When enabled, the scanner switches to pullback-entry mode — it looks for price dipping into the ribbon during an established trend and crossing back out. Toggle this based on your preferred entry style.

Internal HTF Filter

Requires higher timeframe ribbon alignment before any signal fires. Set to None to disable, or choose from 15, 30, 60, 120, 240 minute, Daily, or Weekly. If you're scanning 15-minute charts and set this to 60, only signals that align with the hourly ribbon direction will fire.

Risk Management

All risk management settings from Quantum Ribbon Pro are available in the scanner. Each symbol tracks its own independent trade state — stops, targets, and partial take-profits are calculated and managed per instrument.

Stop Loss Method

Three options: ATR (volatility-adjusted, default), Percentage (fixed % from entry), or Swing (placed at recent price structure). ATR adapts to each symbol's volatility automatically. Swing stops reference actual support/resistance levels and include an ATR buffer beyond the swing point.

Stop Loss Distance

For ATR method: Tight (1.5× ATR), Normal (2× ATR, default), or Wide (2.5× ATR). Tighter stops give better risk/reward but get hit more often. Wider stops stay in trades longer but risk more per trade.

Take Profit Target

Sets the final take-profit at 1.5R, 2R, or 3R. This is the risk multiple — if your stop is 100 points away, 2R targets 200 points of gain.

Partial Take-Profits

Enable this to scale out of positions in three stages instead of closing everything at one target. Configure the percentage to close at each level and the R-multiple for each. Default: close 50% at 1.5R, another 30% at 2R, let 20% run to the final target. The dashboard tracks which partials have hit per symbol.

Break-Even Stop

Automatically moves the stop to entry price once the trade reaches a defined profit level. Trigger options: when TP1 hits, when TP2 hits, or at a custom R-multiple. The dashboard shows (BE) in the position column once break-even is active for a trade.

Ribbon Flip Exit

Closes a position early when the ribbon flips against your direction, before the final take-profit or stop is reached. Only triggers if no partial take-profits have been hit yet. Flip Sensitivity (1–5) controls how many ribbon layers must flip — 5 requires all layers to flip (conservative), lower values exit earlier (aggressive).

Filters & Confirmation

TP RSI Confirmation

Requires all five RSI ribbon pairs to align with the signal direction before it fires. For longs, all five must be bullish. For shorts, all five must be bearish. This reduces signal frequency but adds momentum confirmation. If you've been running Quantum Ribbon Pro with TP RSI as a separate filter, enable this to replicate that setup inside the scanner.

Quantum Range Filter

Blocks signals during choppy, low-volatility conditions by analyzing volatility ratio and ribbon compression. When ATR is low compared to its average and the ribbon is squeezed, the instrument shows Filtered / Range in the Master Trend column and no signals fire. Three sensitivity levels: Low (only blocks extreme chop), Normal (balanced default), High (only trades in clearly trending markets).

S/R Auto-Detection

Restricts signals to moments when price is near a key support or resistance level. The scanner automatically detects pivot highs and lows based on your chosen strength setting — Weak for more levels, Strong for major swings only. The Touch Zone (%) defines how close price must be to a level to qualify. Useful for traders who only want to enter at structure.

Session Filter

Restricts signals to a defined trading window in EST. Set your start and end times to block alerts outside your preferred session. The scanner respects this filter independently — signals only fire during active hours.

Disable Weekend Signals

Prevents signals from firing on Saturday and Sunday. Useful for crypto traders who don't want low-volume weekend setups triggering alerts. The chart still processes and would show the signal visually, but no alerts fire.

Alert System

The scanner fires alerts for every event across all monitored instruments from a single alert setup. You don't need to create separate alerts for each symbol.

To set up alerts: Right-click on the chart, select Add Alert, and choose QR Pro Scanner as the condition. Set it to fire on any alert() call. One alert handles everything.

Alert events include:

  • Entry — new long or short position opened
  • TP1 / TP2 — partial take-profit levels hit
  • Final TP — final take-profit reached, trade closed
  • Stop Loss — stop hit, trade closed
  • Break-Even — stop moved to entry
  • Ribbon Flip Exit — early exit on ribbon reversal

Toggle each event type on or off independently. For example, disable TP alerts if you only want entry and stop notifications.

Alert Formats

Simple — clean single-line message with symbol, timeframe, direction, and event type. Example: [QR Scanner] BYBIT:BTCUSDT.P (15) SHORT ENTRY at 67506.20

JSON — structured data with all price levels. Designed for webhook integrations with bots, Telegram, Discord, or automation platforms. Parses cleanly into any system that accepts JSON payloads.

Custom Template — build your own message using placeholders: {{ticker}}, {{tf}}, {{dir}}, {{type}}, {{price}}, {{sl}}, {{tp}}. Arrange them however your workflow needs.

Deduplication Built In

The scanner includes internal deduplication logic — the same symbol, timeframe, and event type won't fire twice on the same bar even if multiple data requests resolve simultaneously. You won't get duplicate alerts.

Table Appearance

Full control over how the dashboard looks. All color and style settings are in the Scanner Appearance group.

Layout options (in Scanner Layout group):

  • Table Position — choose from top right, bottom right, top left, bottom left, middle right, middle left, or center
  • Table Text Size — tiny, small, normal, or large
  • Update Every N Bars — how frequently the table refreshes (1 = every bar, useful to reduce load on large watchlists)

Color controls: Background color, border color, header background and text color, row text color. Customize status colors for Long, Short, and Wait states. Trend colors for bullish, bearish, and filtered states are adjustable independently.

Important Notes

⚠ Timeframe Rule

The scanner timeframe for each symbol cannot be lower than the chart's current timeframe. If your chart is on 1H, you cannot scan 15M timeframes. Set the chart to a lower timeframe if you need to monitor shorter intervals.

Match Your Settings

For best results, match your scanner settings to your main Quantum Ribbon Pro configuration. If you're trading with TP RSI confirmation enabled on the chart indicator, enable it in the scanner too. This keeps signal logic consistent across both tools.

Mixed market types: Results may vary slightly between different chart symbols since markets operate on different schedules. Crypto runs 24/7 while stocks and forex have defined sessions — this can affect how signals appear across mixed watchlists, especially when using the session filter.

TradingView plan: The scanner uses request.security() calls for each symbol. This works on all TradingView plans, but free plan users may experience slower table updates due to rate limits on data requests.

Standalone use: The scanner works as a standalone tool for traders who want multi-instrument monitoring without opening individual charts. It's not required to have the main Quantum Ribbon Pro indicator on any chart for the scanner to function.
